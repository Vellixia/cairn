# Cairn v0.7.0 Plan

> **Goal:** Make Cairn smarter, more useful, and more visual by adopting proven patterns
> from agentmemory and lean-ctx while keeping the pure-remote client architecture.

## Context

v0.6.4 established the pure-remote client (8 CLI commands, no local engines, all operations
through the server HTTP API). v0.7.0 focuses on **engine intelligence** and **dashboard UX**
to make Cairn genuinely better at helping AI agents, not just functional.

Research basis: deep audit of two reference repos (`agentmemory`, `lean-ctx`) covering
53 MCP tools, 22 agent installers, 4000-line viewer, 56 shell patterns, triple-stream
hybrid search, LLM-driven consolidation, and more.

## Phase 1: Engine Intelligence (server-side)

### 1.1 Anti-inflation cap on reads

**From:** lean-ctx `tools/ctx_read/mod.rs:819-844`

Compressed/framed reads must **never cost more tokens than the raw file**. If the framed
output exceeds the raw token count, ship the raw file verbatim. This is a correctness
invariant that prevents wasting tokens on bad compression.

**Files:** `crates/cairn-context/src/lib.rs` (read function)
**Effort:** Small
**Risk:** None — purely additive guard

### 1.2 Auto-delta on re-read

**From:** lean-ctx `tools/ctx_read/mod.rs:913-1030`

When a file is re-read and has changed on disk since the last read, automatically compute
a diff. If the diff is < 60% of the full token count, send the delta instead of the full
content. Saves tokens on iterative edits.

**Files:** `crates/cairn-context/src/lib.rs`
**Effort:** Medium
**Risk:** Low — diff is fallback, not replacement

### 1.3 Triple-stream hybrid search with graph

**From:** agentmemory `src/state/hybrid-search.ts`

Add a **graph-proximity search stream** to the existing BM25 + vector fusion. Extract
entities from the query, traverse the memory provenance graph (depth 2), score by
`avgEdgeWeight * (1 / pathLength)`. Fuse all three streams via RRF with **dynamic weight
renormalization** when a stream returns empty (prevents BM25-only queries being diluted
by zeroed vector/graph weights).

Add **session diversification**: max 3 results per session, then backfill. Prevents one
long session from dominating recall.

**Files:** `crates/cairn-memory/src/lib.rs` (hybrid_search), `crates/cairn-store/src/helix.rs`
**Effort:** Large
**Risk:** Medium — changes core search ranking

### 1.4 LLM-driven memory consolidation

**From:** agentmemory `src/functions/consolidation-pipeline.ts`

Replace simple tier promotion with **LLM-driven synthesis**:

1. **Semantic tier:** When >= 5 session summaries exist, take the 20 most recent, LLM-merge
   them into typed facts with confidence scores. Dedup against existing semantic memories.
   Increment access count + bump confidence for matches.

2. **Procedural tier:** Extract procedures from recurring patterns (frequency >= 2).
   Dedup by name. Increment frequency + strengthen.

3. **Decay tier:** Decay `strength` by `0.9^decayPeriods` when last accessed > N days ago,
   floored at 0.1.

4. **Reflect tier:** LLM synthesizes higher-order insights from concept clusters
   (graph-based BFS or Jaccard similarity). Reinforce existing insights or create new ones.
   Weekly decay sweep with soft-delete at floor.

**Files:** `crates/cairn-memory/src/lib.rs` (consolidate, crystallize), new `reflect.rs`
**Effort:** Large
**Risk:** Medium — requires LLM calls during consolidation (cost)

### 1.5 Contradiction detection

**From:** agentmemory `src/functions/auto-forget.ts`

When two memories in the same concept group have Jaccard similarity > 0.9, mark the older
one as `isLatest=false`. Prevents stale memories from surfacing alongside their
replacements.

**Files:** `crates/cairn-memory/src/lib.rs`
**Effort:** Medium
**Risk:** Low

### 1.6 Followup-rate retrieval quality metric

**From:** agentmemory `src/functions/smart-search.ts`

Track when an agent re-queries within a window with a **disjoint result set** — a proxy
for "retrieval failed the first time." Expose as a metric on the dashboard and in `/api/metrics`.

**Files:** `crates/cairn-memory/src/lib.rs`, `crates/cairn-api/src/metrics.rs`
**Effort:** Small
**Risk:** None

### 1.7 Bounce tracker + adaptive thresholds

**From:** lean-ctx `core/bounce_tracker.rs`, `core/threshold_learning.rs`

Track when a compressed read was "bounced" (agent re-read the full file after getting a
compressed version). Feed this signal into adaptive mode-selection thresholds so the system
learns which files/modes benefit from compression vs which should just be sent raw.

**Files:** `crates/cairn-context/src/lib.rs`, `crates/cairn-store/src/helix.rs`
**Effort:** Medium
**Risk:** Low

### 1.8 Default-off context injection with opt-in

**From:** agentmemory v0.8.10 lesson

Proactive context injection silently burns ~1000 tokens/turn. Make it **opt-in** via a
preference or env var (`CAIRN_INJECT_CONTEXT=true`). The `UserPromptSubmit` hook checks
this flag before calling `/api/context/assemble`. When off, hooks only do memory
recording (not injection).

**Files:** `crates/cairn-client/src/hook.rs`
**Effort:** Small
**Risk:** None — additive flag

## Phase 2: Dashboard / Web UI

### 2.1 WebSocket live updates

**From:** agentmemory `src/viewer/server.ts`

Replace SSE polling with **WebSocket push**. Register state-change triggers on memory
operations. When memories change, push events to connected dashboard clients.

Auto-reconnect with exponential backoff. Status pill in header (connected/disconnected).

**Files:** `crates/cairn-api/src/events.rs`, `web/src/app/(app)/layout.tsx`
**Effort:** Medium
**Risk:** Low

### 2.2 Cost savings in USD

**From:** agentmemory dashboard

Show cost savings in **dollars** on the dashboard, not just token counts. Formula:
`tokens_saved * $0.30 / 1000`. Display as `$X.XX` or `N cents`.

Add a "Cost Savings" card to the overview alongside the existing "Tokens Saved" headline.

**Files:** `web/src/app/(app)/OverviewContent.tsx`, `crates/cairn-api/src/metrics.rs`
**Effort:** Small
**Risk:** None

### 2.3 Compression Lab

**From:** lean-ctx `cockpit-compression.js` + `/api/compression-demo`

Interactive side-by-side comparison of read modes on real project files. User picks a
file, sees `full | signatures | map | aggressive` rendered side-by-side with token counts.

New endpoint: `GET /api/context/compression-demo?path=...&mode=...` returns the compressed
content + token count for each mode.

**Files:** `web/src/app/(app)/memory/` (new page), `crates/cairn-api/src/lib.rs` (new route)
**Effort:** Medium
**Risk:** None

### 2.4 Architecture report generator

**From:** lean-ctx `routes/graph/architecture.rs`

Auto-generate a **Markdown architecture report** from graph signals: community detection,
god-node detection, import-cycle detection, bridge detection, cohesion metrics.
Downloadable as `.md`.

New endpoint: `GET /api/memory/architecture-report` returns generated Markdown.

**Files:** `crates/cairn-memory/src/lib.rs`, `crates/cairn-api/src/lib.rs`
**Effort:** Large
**Risk:** Medium — graph analysis can be slow on large codebases

### 2.5 Context pressure gauge

**From:** lean-ctx `core/context_ledger.rs`

Sliding-window token budget utilization with **eviction candidates** ranked by phi value.
A real-time context-budget visualization showing how full the context window is and what
should be evicted next.

**Files:** `web/src/components/` (new component), `crates/cairn-api/src/lib.rs`
**Effort:** Medium
**Risk:** Low

### 2.6 Activity heatmap

**From:** agentmemory viewer (GitHub-style contribution grid)

7-row grid, 4 intensity levels, clickable cells. Shows when memories were created/accessed
over time. Visual pattern of agent activity.

**Files:** `web/src/app/(app)/` (new component)
**Effort:** Small
**Risk:** None

### 2.7 Interactive knowledge graph

**From:** agentmemory `src/viewer/index.html` `renderGraph`

Canvas-based force-directed graph with:
- Nodes color-coded by type (decision, fact, gotcha, pattern, preference)
- Curved edges with arrowheads (derived_from, contradicts, supersedes)
- Pan/zoom, hover focus dimming, search highlighting
- Edge labels at zoom thresholds
- Dark mode support
- Sidebar filter by node type

This is already partially in Cairn's `web/src/components/MemoryForceGraph.tsx` but
agentmemory's version is more interactive.

**Files:** `web/src/components/MemoryForceGraph.tsx` (enhance)
**Effort:** Medium
**Risk:** Low

### 2.8 Memory browser with search + filters

**From:** agentmemory `renderMemories`

Memory list with:
- Search input (IME-safe, debounced)
- Type-filter chips (decision, fact, gotcha, note, preference, task)
- Importance badges (color-coded)
- Tag/concept pills
- File-tag pills
- Sortable by: created_at, importance, confidence, access_count

**Files:** `web/src/app/(app)/memory/recall/page.tsx` (enhance)
**Effort:** Medium
**Risk:** None

## Phase 3: API + Error Improvements

### 3.1 Structured REST error envelope

**From:** lean-ctx `http_server/mod.rs:284-290`

Standardize all API errors as:
```json
{ "error": "human-readable message", "error_code": "stable_code" }
```

SDKs and agents branch on `error_code` instead of parsing prose. Already partially done
for 401 responses (v0.6.3 added `reason` field). Extend to all error responses.

**Files:** `crates/cairn-api/src/lib.rs` (ApiError type)
**Effort:** Small
**Risk:** Low

### 3.2 OpenAPI spec endpoint

**From:** lean-ctx `core/openapi.rs`

Add `GET /api/openapi.json` returning a machine-readable OpenAPI spec for all endpoints.
Also add `GET /api/capabilities` for a simpler capabilities discovery.

**Files:** `crates/cairn-api/src/lib.rs`
**Effort:** Medium
**Risk:** None

### 3.3 Query expansion

**From:** agentmemory `src/functions/query-expansion.ts`

LLM generates 3-5 reformulations + temporal concretizations + entity extractions for a
search query. Run triple-stream hybrid search for each reformulation and merge by max
combinedScore.

Add as an optional `expand=true` parameter to `GET /api/search`.

**Files:** `crates/cairn-memory/src/lib.rs`
**Effort:** Medium
**Risk:** Medium — LLM cost per search

## Phase 4: Memory Types + Smart Features

### 4.1 Episodic / procedural / prospective memory split

**From:** lean-ctx `core/episodic_memory.rs`, `core/procedural_memory.rs`

Cairn currently has a single "Memory" type with a `kind` field. Consider promoting to
three distinct storage/recall paths:

- **Episodic:** session-specific events with success/failure stats and success_rate
- **Procedural:** task-conditioned suggestions with confidence, times_used, success_rate,
  activation_keywords
- **Prospective:** future-oriented reminders/todos linked to conditions

**Files:** `crates/cairn-core/src/model.rs`, `crates/cairn-memory/src/lib.rs`
**Effort:** Large
**Risk:** High — changes core data model

### 4.2 Local ML reranker

**From:** agentmemory `src/state/reranker.ts`

Load `Xenova/ms-marco-MiniLM-L-6-v2` via `@xenova/transformers` (in-process, quantized).
Rerank top-20 search results by `query [SEP] title content`. No API call needed.

Gated by `CAIRN_RERANKER_ENABLED=true`.

**Files:** `crates/cairn-embed/src/lib.rs` or new `crates/cairn-rerank/`
**Effort:** Large
**Risk:** Medium — adds ML dependency, increases binary size

### 4.3 Gotcha tracker

**From:** lean-ctx `core/gotcha_tracker/`

Mine recurring pitfalls from sessions into bug memory. Detect patterns where the agent
repeatedly fails at the same type of task and store them as "gotcha" memories that
surface proactively.

**Files:** `crates/cairn-memory/src/lib.rs`, new `crates/cairn-gotcha/`
**Effort:** Medium
**Risk:** Low

### 4.4 56 shell compression patterns

**From:** lean-ctx `core/patterns/` (56 named modules)

Expand Cairn's shell compressor from generic compression to **named pattern modules** for
git, cargo, npm, pnpm, yarn, bun, deno, vite, pip, pytest, mypy, ruff, go, docker, kubectl,
helm, aws, terraform, eslint, prettier, tsc, biome, curl, wget, etc.

Single source of truth registry with `Category` enum (Vcs, Build, PackageManager, Lint,
Infra, Http, Search, FileRead, DirList).

**Files:** `crates/cairn-shell/src/lib.rs`
**Effort:** Large (but incremental — add patterns one at a time)
**Risk:** None

**Files:** New `crates/cairn-a2a/`

**From:** agentmemory `src/functions/actions.ts`, `leases.ts`, `routines.ts`

templates.

**Files:** New crate
**Effort:** Very large
**Risk:** High — significant new subsystem

**From:** lean-ctx `benchmark/locomo/`

**From:** lean-ctx `bench/agent-task/`

18-file agent task benchmark with SWE-bench-style harness. Measures how well the agent
performs with vs without Cairn context.

| **P1.1-1.2** | Anti-inflation cap + auto-delta | Correctness invariants — prevent wasting tokens |
| **P2.2** | Cost savings in USD | Makes ROI tangible — small effort, high visibility |
| **P1.8** | Default-off context injection | Prevents silent token burn — small effort |
| **P1.6** | Followup-rate metric | Retrieval quality signal — small effort |

| **P1.3** | Triple-stream search | Core search quality — large but high impact |
| **P2.6-2.8** | Heatmap + memory browser + graph | Dashboard UX — medium effort |
| **P1.4-1.5** | LLM consolidation + contradiction | Engine intelligence — large, needs LLM |

| **P4.x** | Memory types, reranker, gotchas, patterns | Engine depth — large, incremental |
| **P5-6** | Multi-agent, compliance, benchmarks | Future — very large |

## Non-Goals (explicitly NOT doing in v0.7.0)

- Plugin system with WASM extensions (too complex, no demand yet)
- Buddy/gamification system (cosmetic, not core)
- Formal proofs in Lean 4 (academic, not practical)
- Compliance PDF report generator (enterprise feature, not needed yet)
- FinOps export formats (enterprise feature)
- 18-agent adapter pattern (Cairn supports 3 agents, premature to scale)
- Plugin marketplace manifests (no distribution channel yet)
- Mesh sync for peer-to-peer (centralized server is the architecture)
